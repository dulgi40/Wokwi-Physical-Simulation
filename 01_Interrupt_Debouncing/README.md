  ## 🔌 システム構成および仕様

### 1. 概要
本システムはESP32を使用し、4つのLEDが順次点灯するベースロジックの上で動作します。ユーザーがプッシュボタンを押すと、即座に割り込み（Interrupt）が発生し、緊急制御ロジックが実行されるよう設計されています。また、Wokwiシミュレーション環境内における物理的ノイズであるチャタリング現象の発生有無を実験的に検証しました。

### 2. 主要部品および接続 (Pin Mapping)
| 部品 (Component) | ピン番号 (ESP32) | 役割 (Role) |
|:---:|:---:|:---|
| ESP32 DevKit V4 | - | メインコントローラー |
| LED 1 (Red) | GPIO 12 | 順次点灯用 LED 1 |
| LED 2 (Red) | GPIO 14 | 順次点灯用 LED 2 |
| LED 3 (Red) | GPIO 27 | 順次点灯用 LED 3 |
| LED 4 (Red) | GPIO 26 | 順次点灯用 LED 4 |
| Push Button | GPIO 13 | 割り込みトリガー (Falling Edge) |
| Resistors | 220Ω (x4) | LEDの電流制限および保護 |


  📑 プロジェクトレポート：割り込みとデバウンスの研究

1. きっかけ :
組み込みシステムの核心である割り込み（Interrupt）の動作原理を学習し、プッシュボタンを用いてシステムを緊急制御するロジックを実装しました。

2. 問題提起 :
ボタンのような物理的接点部品で発生するチャタリング（Bouncing）現象が、システムの誤作動や不要な割り込みの重複実行の原因になり得ることを認知し、その解決策を模索しました。

3. 検証 :
Wokwiシミュレーション環境が物理的ノイズをどの程度忠実に再現しているかを確認するため、ソフトウェア的なデバウンスロジックを除去し、millis()値を出力して割り込みの発生間隔を直接追跡しました。

4. 結論 :
検証の結果、Wokwi上では信号が非常に安定して入力されることを確認しました。しかし、これはあくまでシミュレーション上の特性であり、実機ではチャタリングが必ず発生します。したがって、どのような環境でも動作する安定したシステムを構築するためには、「防御的プログラミング」の観点からデバウンス処理が不可欠であることを学びました。

  ## 🔌 시스템 구성 및 사양

### 1. 개요
본 시스템은 ESP32를 사용하여 4개의 LED가 순차적으로 점등하는 베이스 로직 위에서 동작합니다. 사용자가 푸시 버튼을 누르면 즉시 인터럽트(Interrupt)가 발생하여 긴급 제어 로직이 실행되도록 설계되었습니다. 추가적으로 Wokwi 시뮬레이션 환경 내에서 물리적 노이즈인 채터링(Chattering) 현상의 발생 여부를 실험적으로 검증하였습니다.

### 2. 주요 부품 및 연결 (Pin Mapping)
| 부품 (Component) | 핀 번호 (ESP32) | 역할 (Role) |
|:---:|:---:|:---|
| ESP32 DevKit V4 | - | 메인 컨트롤러 |
| LED 1 (Red) | GPIO 12 | 순차 점등 LED 1 |
| LED 2 (Red) | GPIO 14 | 순차 점등 LED 2 |
| LED 3 (Red) | GPIO 27 | 순차 점등 LED 3 |
| LED 4 (Red) | GPIO 26 | 순차 점등 LED 4 |
| Push Button | GPIO 13 | 인터럽트 트리거 (Falling Edge) |
| Resistors | 220Ω (x4) | LED 전류 제한 및 보호 |


📑 프로젝트 리포트: 인터럽트 및 디바운싱 연구

1. 발단 :
임베디드 시스템의 핵심인 인터럽트(Interrupt)의 동작 원리를 학습하며, 푸시 버튼을 이용해 시스템을 긴급 제어하는 로직을 구현했습니다.

2. 문제 제기 :
버튼과 같은 물리적 접점 부품에서 발생하는 채터링(Bouncing) 현상이 시스템 오작동이나 불필요한 인터럽트 중복 실행의 원인이 될 수 있음을 인지하고 해결 방안을 모색했습니다.

3. 실험 및 검증 :
Wokwi 시뮬레이션 환경이 물리적 노이즈를 얼마나 충실히 재현하는지 확인하기 위해, 소프트웨어적 디바운싱 로직을 제거하고 millis() 값을 출력하여 인터럽트 발생 간격을 직접 추적했습니다.

4. 결론 :
실험 결과 Wokwi 시뮬레이션 상에서는 신호가 매우 안정적으로 입력되었습니다. 하지만 이는 시뮬레이션 특성일 뿐이며, 실제 하드웨어(실기)에서는 채터링이 반드시 발생합니다. 따라서 어떤 환경에서도 동작하는 안정적인 시스템을 구축하기 위해서는 '방어적 프로그래밍' 관점에서의 디바운싱 처리가 필수적임을 배웠습니다.
